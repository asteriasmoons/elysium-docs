# /permissions view 

Display all custom command permission settings currently configured in your server. This provides a quick audit of which roles can (or cannot) use what.

<hr className="md-divider-gradient" />

### 💬 Usage
```bash
/permissions view
```
:::info
Use this command to check which roles have special command access — or restrictions — applied through `/permissions set.`
:::

<hr className="md-divider-gradient" />

### 📌 Example
```bash
/permissions view
```
This will:
- Show a full list of commands with customized permissions
- Indicate what role(s) each command is allowed or denied for
- Help you clean up or confirm your configuration

<hr className="md-divider-gradient" />

### 🗃️ Notes
> - This only shows custom permission overrides, not the default access.
> - If no overrides are found, Pandoryx will say so.
> - You must have the Administrator permission to run this command.

<hr className="md-divider-gradient" />

### 🔍 Related Commands
`/permissions set` – Configure who can use each command

`/permissions reset` – Reset everything back to default
