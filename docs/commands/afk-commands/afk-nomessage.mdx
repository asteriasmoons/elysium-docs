# /afk nomessage

Toggle whether sending a message should automatically clear a user’s AFK status. This setting applies server-wide and can only be adjusted by administrators.

<hr className="md-divider-gradient" />

### 💬 Usage
```bash
/afk nomessage
```
:::success
Switches the auto-clear setting for AFK statuses when users send a message.
:::

<hr className="md-divider-gradient" />

### 📌 Example
```bash
/afk nomessage
```
This will:
- Enable or disable automatic AFK removal when users send a message
- Apply to all users in the server
- Notify you of the current toggle status (enabled or disabled)

<hr className="md-divider-gradient" />

### 🗃️ Notes
> - This is an admin-only command and affects everyone’s AFK behavior in the server.
> - When enabled: AFK status is removed as soon as a user sends a message.
> - When disabled: Users remain AFK until they explicitly use `/afk disable`.

<hr className="md-divider-gradient" />

### 🔍 Related Commands
`/afk enable` – Set your AFK status with a message 

`/afk disable` – Remove your AFK status manually
